What is clustering?

Cluster analysis or clustering is the task of grouping a set of objects in such a way that objects in the same group (called a cluster) are more similar (in some sense) to each other than to those in other groups (clusters).

Related terms

  • Unsupervised learning

General goal

  • Group similar objects together
  • Why?
    • Help us find patterns in the data
    • Simplify the data

Types of data

When applying clustering to a Cytoscape network there are two basic approaches:

  1. clustering based on the network topology. The idea is that nodes that are close to each other should in the same cluster and nodes that are further away should be in different clusters. Typically, the goal is to maximize some metric that measures inter-cluster distance vs. intra-cluster distance. MCL and MCODE are examples of clustering based on network toplogy.
  2. clustering based on attributes. In this instance, the idea is that similarity is not measured based on topology, but on the similarity of the attributes of the nodes or edges. The best examples of attribute-based clustering are hierarchical and k-means.

Clustering in Cytoscape

There are several Cytoscape apps that perform clustering. Some are pretty general and some are quite specific for certain types of data. Here are some of the most popular apps:

MCODE
Clusters a given network based purely on topology to find densely connected regions. Does not support any kind of edge weighting
clusterMaker2
App that supports attribute clustering, clustering based on topology, cluster ranking, and dimensionality reduction. clusterMaker2 currently supports 41 different algorithms
cytoHubba
Predicts and explores important nodes and subnetworks in a given network by several topological algorithms.
