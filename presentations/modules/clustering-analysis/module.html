What is clustering?

Cluster analysis or clustering is the task of grouping a set of objects in such a way that objects in the same group (called a cluster) are more similar (in some sense) to each other than to those in other groups (clusters).

Related terms

  • Unsupervised learning
  • Community detection
  • Automatic classification
Key thing is no training, known labels, or gold standard

General goal

  • Group similar objects together
  • Why?
    • Help us find patterns in the data
    • Simplify the data

Types of data

When applying clustering to a Cytoscape network there are two basic approaches:

  1. clustering based on the network topology. The idea is that nodes that are close to each other should in the same cluster and nodes that are further away should be in different clusters. Typically, the goal is to maximize some metric that measures inter-cluster distance vs. intra-cluster distance. MCL and MCODE are examples of clustering based on network toplogy.
  2. clustering based on attributes. In this instance, the idea is that similarity is not measured based on topology, but on the similarity of the attributes of the nodes or edges. The best examples of attribute-based clustering are hierarchical and k-means.

Types of clustering algorithms

agglomerative: 
bottom-up approach that pairs the "closest" two items, and merges that with the next closest item, etc.
centroid-based: 
iteratively calculates the centroid of a set of clusters and then refines them, adding or removing items from clusters to maximize or minimize some metric (e.g. within-cluster variances). Centroid-based approaches typically require an initial value for the number of clusters.
modularity-based: 
similar in principle to centroid-based clustering, but the goal is to optimize modularity.
topology-based: 
a wide variety of algorithms that explicitly use the topology of the graph to cluster it. This might include flow-based approaches, message passing, density metrics, or matrix measures such as eigenvalues.
*

Types of clustering algorithms

biclusters: 
simultaneous clustering of rows and columns of a matrix, which yields a subset of rows (or columns) that exhibit similar behavior scross a subset of columns (or rows). The result is often portrayed as a heat map with distinctive blocks the represent the biclusters.
fuzzy clusters: 
typical cluster algorithms assign each individual item to a cluster. In a fuzzy cluster, items can be assigned to multiple clusters, usually depending on some metric of "closeness" to the cluster.
dimensionality reduction: 
not really a clustering algorithm, but still extremely useful and often used as a visualization aid for grouping values. Dimensionality reduction techniques attempt to reduce the number of dimensions of a data set and plot them or display them in some way.
* include dimensionality reduction as a cluster "type"

Clustering in Cytoscape

There are several Cytoscape apps that perform clustering. Some are pretty general and some are quite specific for certain types of data. Here are some of the most popular apps:

MCODE  
Clusters a given network based purely on topology to find densely connected regions. Does not support any kind of edge weighting
clusterMaker2  
App that supports attribute clustering, clustering based on topology, cluster ranking, and dimensionality reduction. clusterMaker2 currently supports 41 different algorithms
cytoHubba  
Predicts and explores important nodes and subnetworks in a given network by several topological algorithms.
TiCoNE  
TiCoNE (Time Course Network Enricher) is a tool for the combined analysis of time series expression data together with biological networks.
CytoCluster  
a Cytoscape app for analysis and visualization of clusters from network
