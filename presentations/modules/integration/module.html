Cytoscape as an Integration Platform

Cytoscape is designed to support integration of multiple data types

  • Tabular data from
    • Excel
    • tsv, csv, or txt
    • web services
  • Network data from
    • XGMML, SIF, cx, cx2, BioPax, GraphML, Cytoscape JSON, PSI-MI, GML, SBML
    • Excel, tsv, csv, txt, web services

Cytoscape as an Integration Platform

Cytoscape has no inherent knowledge about biology

  • Doesn't care about network source
  • Doesn't care about attribute sources
  • Only cares about nodes, edges, networks, tables
  • Table columns can be integer, long integer, floating point, strings, boolean, or lists of any of these

Typical Integration Workflow

  1. Import or create a network
    • From a web service app: e.g. STRING, IntAct
    • From a file (e.g. AP/MS output)
    • From an analysis (e.g. scNetViz, GeneMANIA
  2. Add additional data as attributes
    • From Excel or a text file
  3. Analyze the data
    • Clustering, network analysis, enrichment
  4. Visualize the results
    • Color, shape, size, heatmaps, etc.
  5. Possibly integrate more data
    • No a priori limit to the number of columns
    • Make sure column names don't overlap, though
    • Go back to step 3

Common Integration Issues

  • Incorrect key mapping
    • When you are importing a table, make sure to correctly choose the column in the existing network and the column in your table to act as matching keys
    • Not all networks have name as the best choice for a matching key
    • In the case of stringApp, for example, display name, or often query term is the best column to choose
  • Getting the order wrong
    • Generally, you always want to create your network first
    • Then you can integrate tabular data
    • Some apps create networks based on tabular data (e.g. scNetViz), but that is the exception
